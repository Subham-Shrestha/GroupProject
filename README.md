## 📋 Gym Membership Management System
A collection of C-based utility programs designed to manage gym membership records using persistent file storage. This project demonstrates core concepts of structural data organization and file handling within the C programming language.

## 🏗️ Project Architecture
The system is composed of three specialized modules, each handling a distinct part of the record lifecycle:

### 1. Data Entry & Storage (Recordss.c)
This module serves as the primary input interface.

Structural Organization: Utilizes nested structures to store complex data including Name (First/Last), Date (Join Date), and Member details.

Dynamic Fee Calculation: Features a logic-based fee generator that assigns costs based on "Premium" status (1500 for premium, 500 for standard).

File Output: Writes formatted data directly into members.txt for permanent storage.

### 2. Formatted Data Retrieval (RecordsRead.c)
A specialized viewer that transforms raw file data into a human-readable format.

Clean Formatting: Uses field-width specifiers (e.g., %-10s) to ensure columns align perfectly in the terminal.

Header Logic: Smartly skips metadata and header lines in the text file to process only relevant member entries.

### 3. Categorical Filtering (Category.c)
A search and export utility for organizing members by their subscription type.

Interactive Filtering: Allows users to input a specific plan (Weekly, Monthly, or Yearly) to isolate specific records.

Dual Output: Displays filtered results in the terminal while simultaneously generating a secondary report file named planType.txt.

Data Parsing: Employs sscanf and strcmp to accurately parse string-based data from the storage file.

##🛠️ Technical Implementation
This project implements several advanced C programming techniques:

Nested Structures: Groups related data types to represent real-world objects.

File Stream Handling: Manages multiple file pointers for simultaneous reading and writing.

String Logic: Implements string comparison and formatted buffer reading.

Formatted I/O: Precise control over terminal and file output alignment.

## 🚀 Getting Started
Prerequisites
A C compiler (e.g., GCC, Clang).

Standard C libraries (stdio.h, string.h).

Execution Steps
Initialize Records:

# compile
gcc Recordss.c -o records
./records
View All Members:

# compile
gcc RecordsRead.c -o read
./read
Filter by Plan:

# compile
gcc Category.c -o filter
./filter

## 📂 File Structure
Recordss.c: Source code for adding members.

RecordsRead.c: Source code for displaying the member table.

Category.c: Source code for filtering and reporting.

members.txt: The primary database file generated by the system.

planType.txt: The filtered report file generated during searches.
